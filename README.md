# Google-Playstore-EDA

# About The Project
This is my beginner-level Data Analysis project where I performed Exploratory Data Analysis (EDA) on the Google Play Store dataset.
In this project, I worked with real-world app data to understand trends, ratings, reviews, installs, and categories. Since this is one of my early projects in Data Analysis, my main goal was to understand how real datasets look and how to clean and analyze them step by step.
This project helped me gain confidence in working with messy data and turning it into meaningful insights.

# Project Goal

The main goals of this project were:
Clean the dataset properly
Handle missing values and incorrect data types
Understand app performance based on ratings and installs
Analyze which categories are most popular
Study the relationship between reviews, ratings, and installs
Practice data visualization

# Tools & Libraries Used

I used the following tools and libraries:
Python
Pandas
NumPy
Matplotlib
Seaborn

# Analysis Performed
Some of the analysis I performed includes:
Distribution of App Ratings
Apps by Category
Total Installs by Category
Free vs Paid Apps Analysis
Relationship between Reviews and Installs
Size and Price Analysis
Most Popular App Categories

# Key Insights
From this analysis, I found:
Which app categories have the highest number of installs
How ratings are distributed across apps
The relationship between number of reviews and installs
Which categories are more competitive
Difference between Free and Paid apps

# What I Learned From This Project
Through this project, I learned:
How to explore a new dataset
How to clean messy real-world data
Handling missing values and incorrect formats
Using groupby() in pandas
Creating meaningful visualizations
Thinking step by step like a data analyst
